# 採用ライブラリ一覧

## Frontend
1. **React 18**
   - 役割: UIフレームワークおよび全体の State 管理 (useReducer)。
   - 理由: 宣言的 UI による管理体制が構築しやすいため。
   - 代替案の棄却理由: Vue / Svelte なども強力だが、エコシステムの大きさと要件の安定性から採用。
2. **Fabric.js**
   - 役割: HTML5 Canvas ベースの 2D 描画とインタラクション。
   - 理由: オブジェクト指向の 2D Canvas 描画システムとして最も実績があり、単純なマウスドラッグ処理なども容易に記述できる。
   - 代替案の棄却理由: Konva.js や PixiJS についても検討したが、CAD 的な直線や選択UIの提供において Fabric.js のプリミティブが使いやすいため。
3. **Vite**
   - 役割: フロントエンドビルドツールおよび開発サーバー。
   - 理由: クイックな立ち上がりと設定の少なさ。
4. **Playwright**
   - 役割: ブラウザの自動化と E2E テスト・スクリーンショット保存。
   - 理由: 独立した環境で素早くスクリプト化でき、安定性が高いため。

## Backend
1. **FastAPI**
   - 役割: API サーバー。
   - 理由: Pythonベースで高速、かつ Pydantic による JSON のバリデーションサポートが強力であるため。
   - 代替案の棄却理由: Flask / Django は機能が少なすぎるか過剰である。
2. **ezdxf**
   - 役割: DXF ファイルのパースおよび書き出し。SVG / PDF のエクスポート (drawing add-on)。
   - 理由: Python で DXF を扱う際のデファクトスタンダードであり、アドオンを利用した描画エクスポートも充実している。
   - 代替案の棄却理由: カスタムパーサ作成 (非目標でありコスト高)。
3. **Pydantic**
   - 役割: 型バリデーションと JSON スキーママッピング。
4. **Uvicorn**
   - 役割: ASGI サーバー。
